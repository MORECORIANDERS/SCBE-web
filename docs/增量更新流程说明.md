# 可转债数据增量更新流程

## 📋 数据库表结构

| 表名 | 用途 | 主键 | 增量更新策略 |
|------|------|------|------------|
| `bond_list` | 可转债代码基础信息表 | `bond_code` | 每日增量更新，新增转债自动入库 |
| `bond_snapshot` | 每日行情快照 | `(trade_date, bond_code)` | 每日覆盖更新 |
| `bond_static` | 可转债详细信息（F10） | `bond_code` | 按需更新，首次全量 |

```bash
python scripts/fetch_and_save_snapshot.py
```

**Step 2: 更新可转债列表（可选，有新增转债时）**
```bash
# 自动更新 bond_list 表（新增转债会入库，已有转债会更新时间戳）
# fetch_and_save_snapshot.py 已包含此功能
```

**Step 3: 更新 F10 详细数据（可选，按需执行）**
```bash

```

### 模式二：首次全量初始化

## 💡 增量更新原理

### bond_list 表的增量逻辑

```sql
INSERT INTO bond_list (bond_code, bond_name, market, is_active)
VALUES (?, ?, ?, 1)
ON DUPLICATE KEY UPDATE
    bond_name = VALUES(bond_name),  -- 更新名称（可能变化）
    updated_at = CURRENT_DATE       -- 更新时间戳
```

- **新增转债**: INSERT 成功，自动入库
- **已有转债**: UPDATE 更新名称和时间戳
- **不变化的字段**: 保持不变

### bond_snapshot 表的增量逻辑

```sql
INSERT INTO bond_snapshot (trade_date, bond_code, ...)
VALUES (?, ?, ...)
ON DUPLICATE KEY UPDATE
    price = VALUES(price),           -- 更新当日行情
    change_pct = VALUES(change_pct),
    ...
```

- **同一日期**: 覆盖更新当日行情
- **不同日期**: 新增记录

### bond_static 表的增量逻辑

```sql
INSERT INTO bond_static (bond_code, bond_name, ...)
VALUES (?, ?, ...)
ON DUPLICATE KEY UPDATE
    ... = VALUES(...)  -- 所有字段都可能更新
```

- **首次入库**: 全量写入
- **后续更新**: 增量更新变化的字段

## ⏰ 执行时机建议

| 数据类型 | 更新频率 | 推荐时间 | 脚本 |
|---------|---------|---------|------|

## 🔧 常见问题

**Q: 为什么 F10 数据不需要每日更新？**
A: F10 数据（转股价、评级、发行规模等）相对稳定，不需要每日更新。建议每周或每月更新一次。

**Q: 如何处理新增的可转债？**
A: `fetch_and_save_snapshot.py` 会自动处理。当日新上市的可转债会在新浪接口中出现，自动写入 `bond_list` 表。

**Q: 如何恢复误删除的数据？**
A: 重新运行对应脚本即可。`ON DUPLICATE KEY UPDATE` 会恢复数据。

**Q: 如何清空并重新初始化？**
A: 需要手动删除表后重新初始化：
```sql
DROP TABLE IF EXISTS bond_list;
DROP TABLE IF EXISTS bond_snapshot;
DROP TABLE IF EXISTS bond_static;
```

然后按照"首次全量初始化"流程重新执行。
