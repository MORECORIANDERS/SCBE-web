项目整体架构

1. 数据采集：Python + mootdx 抓取全市场可转债基础信息、每日行情数据

2. 定时调度：腾讯CloudBase 云函数 + 定时触发器，A股交易日自动执行采集

3. 数据存储：放弃CloudBase自带MongoDB，改用 MySQL 8.0 关系型数据库

4. 后端服务：CloudBase云函数编写HTTP接口，提供数据查询能力

5. 前端可视化：Vue3 + Vite + Ant Design Vue + ECharts，实现数据展示、筛选

6. 整体流程：mootdx采数 → CloudBase云函数清洗入库MySQL → 云函数提供API → Vue前端渲染展示

- 前端：Vue3 + Vite + Ant Design Vue 4.x + ECharts + Axios
- 部署：前端打包后部署到CloudBase静态托管
- 一、数据采集模块需求
- 1. 使用mootdx自动优选通达信最佳服务器，稳定获取沪深全量可转债数据
- 2. 采集两类数据：
- \- 转债基础信息：代码、名称、正股代码/名称、行业、上市日期、到期日期、面值、转股价、发行规模、信用评级
- \- 每日行情数据：交易日、开盘价、收盘价、最高价、最低价、成交量、成交额、转股溢价率、剩余规模、正股价格、正股涨跌幅
- 3. 数据清洗：处理空值、异常值，数值保留3位小数
- 4. 入库规则：
- \- 基础表：存在则更新，不存在则插入
- -行情表：转债代码+交易日 联合唯一，自动去重，避免重复入库
- 5. 云函数配置：内存512M、超时30秒，配置A股交易日定时Cron触发6. 禁止使用Cloud Run（海外IP无法连接通达信服务器），全程使用国内云服务
- 二、MySQL数据库设计（严格按此建2张表）
- 表1：bond\_info 可转债基础信息表
  字段：转债代码(主键)、名称、正股代码、正股名称、行业、上市日期、到期日期、面值、转股价、发行规模、信用评级、创建时间、更新时间要求：主键唯一，建立行业、正股代码普通索引，时间自动更新
- 表2：bond\_daily 可转债每日行情表
  字段：自增ID、转债代码、交易日、开盘/最高/最低/收盘价、成交量、成交额、转股溢价率、剩余规模、到期收益率、双低数值、正股价格、正股涨跌幅、创建时间要求：
  1. 所有价格、比率、规模使用  decimal  类型，杜绝浮点精度丢失2. 建立  code+trade\_date  联合唯一索引，防止重复数据3. 建立交易日、双低、溢价率常用查询索引，适配前端排序筛选
