# 可转债CCI+WR双超卖检测云函数

## 📋 功能说明

每天定时检测可转债市场，当某只转债同时满足以下条件时，通过飞书机器人发送通知：

- **CCI < -100**（CCI超卖）
- **WR > 80**（威廉指标超卖）

## 🔧 技术指标计算

### CCI指标（顺势指标）
```
TYP = (HIGH + LOW + CLOSE) / 3
CCI = (TYP - MA(TYP, 14)) / (0.015 * AVEDEV(TYP, 14))
```

### WR指标（威廉指标）
```
WR = (HHV(HIGH, 14) - CLOSE) / (HHV(HIGH, 14) - LLV(LOW, 14)) * 100
```

### 超卖条件
- CCI < -100：价格低于其平均价格较大程度
- WR > 80：近期价格处于相对低位
- **共振超卖**：两者同时满足，说明技术面超卖信号强烈

## 🚀 部署步骤

### Step 1: 安装 CloudBase CLI（如果还没有）

```bash
npm install -g @cloudbase/cli
```

### Step 2: 登录 CloudBase

```bash
tcb login
```

### Step 3: 创建云函数并部署

```bash
# 进入函数目录
cd cloudfunctions/cb_oversold_detector

# 安装依赖
npm install

# 部署到云端
tcb functions:deploy cb_oversold_detector
```

### Step 4: 配置定时触发器

在 CloudBase 控制台配置触发器：

1. 访问 https://console.cloud.tencent.com/tcb
2. 选择环境 → 云函数 → 函数管理
3. 找到 `cb_oversold_detector`
4. 点击"触发方式" → "添加触发器"
5. 配置触发周期：
   - **触发周期**：定时触发
   - **Cron 表达式**：`0 0 19 * * * *`（每天晚上19:00）
   - **说明**：每晚7点执行，收盘后3.5小时

## ⏰ 触发时间

| 时间 | 说明 |
|------|------|
| 19:00 | 每日收盘后（15:00）约4小时执行 |
| 理由 | 此时段数据较为稳定 |

## 🔍 数据流程

```
1. 从数据库 bond_list 获取活跃转债列表
   ↓
2. 遍历每只转债，从新浪获取日K线数据（最近20天）
   ↓
3. 计算CCI和WR指标
   ↓
4. 筛选满足双超卖条件的转债
   ↓
5. 发送飞书通知（按CCI升序，最超卖的在前）
```

## ⚙️ 配置参数

| 参数 | 值 | 说明 |
|------|-----|------|
| CCI_PERIOD | 14 | CCI计算周期 |
| WR_PERIOD | 14 | WR计算周期 |
| CCI_THRESHOLD | -100 | CCI超卖阈值 |
| WR_THRESHOLD | 80 | WR超卖阈值 |
| MAX_NOTIFY | 20 | 通知最多显示数量 |

## 📝 注意事项

1. **数据源**：使用新浪财经日K线API
2. **执行时间**：约2-3分钟（350只转债 × 100ms延迟）
3. **错误处理**：部分转债获取失败不影响整体流程
4. **通知频率**：只在有满足条件的转债时发送

## 🔧 常见问题

**Q: 为什么使用新浪API而不是mootdx？**
A: 新浪API无需连接通达信服务器，稳定性更高，适合云函数环境。

**Q: 如何调整检测灵敏度？**
A: 修改 `index.js` 中的阈值参数：
```javascript
const CCI_THRESHOLD = -100;  // 降低绝对值（如-80）会更敏感
const WR_THRESHOLD = 80;     // 降低数值（如70）会更敏感
```

**Q: 可以同时监控其他指标吗？**
A: 可以，在 `index.js` 中添加更多指标计算逻辑即可。
